## 機能

1. PostgreSQLサービスの停止
   - Docker Composeを使用してサービスを停止
   - コンテナの停止を確認

## 使用方法

```bash
./syllabus.sh -p stop
```

## 処理の流れ

1. PostgreSQLの停止
   - `docker-compose down`の実行
   - コンテナの停止を確認

## 依存関係

- Docker
- Docker Compose
- PostgreSQLコンテナ

## 注意事項

- 実行前にPostgreSQLサービスが起動していることを確認
- コンテナ名は`postgres-db`を使用
- データベースの接続を確認してから停止することを推奨
